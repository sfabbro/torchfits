## 2026-03-17 - Python String Operation Bottlenecks in Tight Loops
**Learning:** When parsing large quantities of text data, such as FITS headers mapping string subsets to Python types, splitting the code across many small static or classmethods introduces significant function call overhead. Additionally, repeated string slicing and operations like `find` combined with character-by-character checks inside multiple function layers stack up significantly. A surprising amount of time in `FastHeaderParser.parse_header_string` was just function overhead and intermediate allocations for small string subsets.
**Action:** When optimizing text parsers that deal with millions of small segments (like 80-char FITS cards), consider inlining critical path methods (`_parse_value`, `_parse_string_value`, `_find_comment_separator`) directly into the main loop to avoid Python function call overhead. Use fast early exits (`idx = value_comment.find("/"); if idx == -1 or "'" not in value_comment[:idx]`) to bypass complex state machine logic when it's not needed. Avoid allocating strings in the hot path.
